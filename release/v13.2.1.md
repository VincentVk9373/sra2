# Release v13.2.1

**Shadowrun Anarchy 2 — FoundryVTT System**

Cybercombat update: cyberdeck/vehicle damage trackers on the character sheet, auto-resolve against ICE, cyber defense/counter-attack between deckers and technomancers, biofeedback routing based on connection mode, and removal of the spell zone/damage distribution system.

---

## New Features

### Cyberdeck Damage Tracker on Character Sheet
- Inline damage track under each cyberdeck: Light (L, yellow), Severe (S, orange), and Incapacitating (I, red) checkboxes
- Thresholds based on effective Firewall (FW, FW x2, FW x3)
- An incapacitated cyberdeck is struck through on the sheet (line-through + reduced opacity)

### Firewall (FW) and Attack (ATT) Malus
- `firewallMalus` and `attackMalus` fields on cyberdecks
- +/- buttons directly on the sheet to adjust malus during play
- Displays effective value and delta in orange when a malus is active
- Damage thresholds and VD use effective values

### Connection Lock
- Lock/unlock toggle visible only in cold-sim and hot-sim modes
- Red padlock icon with pulsing glow animation when locked
- Green running icon with "Free" status text when unlocked
- Status text ("Locked" / "Free") displayed before the icon

### Cyber Attack Row
- "Cyber Attack" sub-row with dice pool (Piratage/Cybercombat + Willpower), RR and VD
- Supports bookmarking for quick access

### Vehicle/Drone Damage Tracker
- Inline damage track on the character sheet for each vehicle/drone
- Bidirectional update: modifying damage from the character sheet updates the drone and vice-versa

### Cybercombat vs ICE — Auto-Resolve
- Cyberdeck attacks and complex forms against ICE are auto-resolved (no defense roll)
- ICE uses the server index as fixed successes
- Three possible outcomes:
  - **Decker wins**: damage = VD + net successes, button to apply damage to the ICE
  - **Draw**: no damage on either side
  - **ICE wins**: effects based on ICE type:
    - **Black**: biofeedback damage (VD + net successes)
    - **Killer**: matrix damage (VD + net successes)
    - **Blocker**: ATT reduction, apply button
    - **Acid**: FW reduction, apply button
    - **Blaster / Glue**: connection lock, apply button

### Cybercombat Decker vs Decker / Technomancer
- **Cyber Defense** and **Cyber Counter-attack** buttons (replacing standard physical combat buttons)
- Auto-detection of defender type:
  - **Decker**: Defense = Piratage/Cybercombat + cyberdeck FW, Counter-attack = Piratage/Cybercombat + Willpower
  - **Technomancer**: Defense = Technomancie/Complex Forms + Willpower, Counter-attack = Technomancie/Complex Forms + Resonance

### Complex Forms = Same Cybercombat Behavior
- Auto-resolve against ICE (identical to cyberdeck)
- Cyber Defense / Cyber Counter-attack buttons against other matrix actors
- Damage type: `matrix` (instead of `physical`)
- Excluded from `isSpellDirect` and `isSpellIndirect` processing (no zone/damage distribution)

### Conditional Matrix Thresholds
- Matrix threshold row on the character sheet only shown when an active cyberdeck or "emerged" trait is present
- Character's matrix thresholds equal the active cyberdeck's thresholds (FW, FW x2, FW x3)

## Bug Fixes

### Biofeedback Routing Fixed
- **Cold-sim / Hot-sim**: damage applied to character gauge (resisted by Willpower)
- **RA / offline**: damage applied to active cyberdeck (resisted by Firewall)
- Biofeedback filter (cyberdeck program) grants +1 to mental thresholds in VR

### Matrix Damage Routed to Cyberdeck
- `matrix` damage applied to the active cyberdeck's `cyberdeckDamage` track (not the character's `matrixDamage` gauge)
- "Emerged" technomancers still receive matrix damage on their personal gauge

### Complex Form Cost
- Fixed cost calculation that was doubling (5000 + rating x 5000 instead of fixed 5000)

### Handlebars Scope in Damage Trackers
- Fixed invalid `../../_id` and `../../uuid` references in nested `{{#each}}` loops

## Improvements

### Yen Display Formatting
- The yen amount input on the character sheet now displays formatted numbers (e.g. `12 500` instead of `12500`)
- Uses the same `formatNumber` helper as feat costs
- Formatted input is parsed back to a number on save

---

## Removals

### Spell Zone/Damage Distribution System
- Complete removal of the interactive success distribution widget between damage and zone (direct and indirect spells)
- Removed JavaScript handler, CSS, `damageSuccesses` field, and associated translations
- Attack successes are now used directly without distribution

### Legacy Damage Gauges (`matrixDamage`, `magicDamage`)
- Removed `matrixDamage` and `magicDamage` SchemaFields from the character data model
- Matrix damage is now routed to the active cyberdeck's `cyberdeckDamage` track
- All damage (physical, mental, biofeedback) uses the single `damage` gauge on the character
- Cleaned up all references in sheet-helpers (regex, path resolution, checkbox processing)
