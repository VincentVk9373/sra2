# Release v13.2.0

**Shadowrun Anarchy 2 — FoundryVTT System**

Mise a jour majeure du systeme matriciel : nouvel acteur Serveur, programmes cyberdecks, mode de connexion sur le personnage, routage des degats de biofeedback selon le mode RV, et traduction des types de GLACE. Ajout des vehicules/drones personnalises, du mode de controle, et du glisser-deposer pour creer des contacts.

---

## Nouvelles fonctionnalites

### Acteur Serveur (closes #93)
Un nouveau type d'acteur "Serveur" permet au MJ de gerer les serveurs matriciels et leurs GLACE.

- **Indice de serveur** (2-12) avec calcul automatique du Firewall
- **Securite physique** : option checkbox qui ajoute +1 a l'indice effectif
- **Roster de GLACE** : les acteurs GLACE sont lies au serveur par glisser-deposer depuis la sidebar
- **Deploiement** : bouton "Deployer une ICE" qui place aleatoirement un token GLACE a cote du token serveur sur la scene (8 emplacements : N, NE, E, SE, S, SW, W, NW)
- **Glisser-deposer vers la scene** : les GLACE peuvent etre draggees depuis la fiche serveur vers le canvas
- **Suivi de l'etat** : les GLACE deployees (avec token sur la scene) sont distinguees des GLACE en attente
- **Indice du token** : lors du deploiement, le `serverIndex` est applique sur le delta du token (l'acteur template GLACE n'est pas modifie)
- **Ouverture de fiche** : le bouton "Ouvrir" sur une GLACE deployee ouvre la fiche du token (acteur synthetique), pas l'acteur de base
- **Alerte Patrouilleuse** : un avertissement s'affiche si aucune GLACE de type Patrouilleuse n'est dans le roster
- **Rafraichissement automatique** : la fiche serveur se met a jour automatiquement quand un token GLACE est cree ou supprime sur la scene
- **Bouton refresh** : disponible en complement pour forcer la synchronisation

### Mode de connexion sur le personnage (closes #47)
Le mode de connexion (RA/RV) est desormais un attribut du personnage, pas du cyberdeck.

- **4 modes** : Deconnecte, RA, RV Cold-sim, RV Hot-sim
- **Badge dans le header** : visible en mode reduit, positionne en bas a droite du header, cliquable pour changer de mode
- **Selecteur dans la fiche** : egalement disponible dans la section etendue du personnage
- **Affichage des effets** : sous le selecteur, les effets mecaniques du mode actif sont affiches :
  - Cold-sim : action cybercombat gratuite, biofeedback etourdissant, choc d'ejection leger
  - Hot-sim : action cybercombat gratuite, avantage cybercombat, biofeedback physique, choc d'ejection grave
- **Style visuel** : le badge est bleu cyan (RA/Cold-sim), rouge (Hot-sim), ou gris fonce (Deconnecte)

### Programmes Cyberdeck
Trois programmes peuvent etre actives sur un atout cyberdeck, chacun ajoutant +1 au niveau recommande :

- **Biofeedback** : permet d'infliger des degats de biofeedback au lieu de degats matriciels
- **Filtre de biofeedback** : procure 1 point d'Armure contre les degats de biofeedback
- **Verrouillage de connexion** : empeche une cible en RV de se deconnecter pendant un tour

Les programmes actifs s'affichent en sous-lignes sous l'atout cyberdeck sur la fiche personnage (meme style que les capacites Emerge).

### Routage des degats de biofeedback
Nouveau type de degat `biofeedback` avec routage automatique selon le mode de connexion :

- **RA / Deconnecte** : les degats de biofeedback s'appliquent a l'appareil (degats matriciels), resistes par le Firewall
- **RV Cold-sim / Hot-sim** : les degats de biofeedback s'appliquent au personnage, resistes par la Volonte (seuils mentaux)
- La GLACE Noire (Black ICE) utilise desormais le type `biofeedback` au lieu de `physical`

### Avantage automatique en Hot-sim
Quand un personnage en mode Hot-sim lance un test de Piratage ou de Cybercombat, le mode de jet est automatiquement mis en "avantage" (seuil de succes 4+ au lieu de 5+). Cet avantage est annule si le personnage a une blessure grave (le desavantage prend le dessus).

### Vehicules/Drones personnalises (closes #50)
Les vehicules et drones peuvent desormais etre crees avec des statistiques entierement personnalisees.

- Nouveau type de vehicule "Vehicule personnalise" avec champs editables : Autopilote, Structure, Maniabilite, Vitesse, Vitesse de vol, Blindage, Monture d'arme
- Les stats personnalisees sont utilisees a la place des presets quand le type "custom-vehicle" est selectionne

### Mode de controle des vehicules/drones (closes #23)
Chaque vehicule/drone dispose desormais d'un mode de controle :

- **Autonome** : le drone agit seul avec son Autopilote
- **Manuel** : conduit physiquement par un personnage
- **En plongee** : le rigger controle le drone via la Matrice
- **Fauteuil du capitaine** : controle assiste

Le mode est affiche sur la fiche personnage a cote du nom du vehicule et modifiable via le menu contextuel.

### Creation de contact par glisser-deposer (closes #20)
Glisser-deposer un acteur PNJ sur la fiche d'un personnage cree automatiquement un atout Contact avec le nom de l'acteur.

## Corrections de bugs

### Degats mentaux appliques sur la jauge de degats (closes #90)
Les degats de type "mental" (sorts directs, etc.) s'appliquent desormais correctement sur la jauge de degats physiques du personnage avec les seuils de Volonte, au lieu d'etre ecrits sur un champ `magicDamage` invisible.

### Sauvegarde de la fiche GLACE (closes #93)
Corriges les problemes de nom et d'image qui revenaient aux valeurs par defaut lors du changement de type ou d'indice de serveur. Les handlers soumettent desormais le formulaire complet au lieu de faire un `update()` partiel.

### Decochage des cases de degats sur GLACE
Les cases de degats sur la fiche GLACE peuvent desormais etre decochees visuellement. Le probleme venait de la classe CSS `checked` qui restait apres le decochage de l'input HTML. Un handler soumet le formulaire au changement de checkbox.

### Correction du traitement des checkboxes de degats
Quand toutes les cases de degats sont decochees, le formulaire HTML n'envoie aucune donnee pour ce champ. L'ancien code conservait les valeurs precedentes (cochees). Le nouveau code detecte l'absence du champ et met explicitement toutes les valeurs a `false`.

### Traduction des types de GLACE (closes #97 partiellement)
Les types de GLACE (Patrouilleuse, Acide, Blaster, etc.) utilisent desormais des cles de traduction i18n au lieu de chaines francaises hardcodees. Supporte FR et EN.
