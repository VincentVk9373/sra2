# Release v13.3.0

## Slug system overhaul (unique identifiers)

Skills and specializations now use **slugs** (language-independent unique identifiers) instead of localized names. This fixes numerous matching bugs, notably specializations no longer appearing under their parent skills.

### Automatic migration
A migration (13.2.4) runs automatically on load. It:
- Adds slugs to all existing skills (actors, world items, compendiums)
- Adds slugs to all existing specializations
- Converts specialization `linkedSkill` from localized names to slugs
- Converts feat RR targets (`rrTarget`) from localized names to slugs
- Fixes `narrationActions` inherited incorrectly from the old default (see #111)

### Centralized constants
- `SKILL_SLUGS` and `SPEC_SLUGS` in `constants.ts` — no more hardcoded slugs anywhere in the code
- `SKILL_NAME_TO_SLUG` for name <-> slug resolution

## Bug fixes

### Specializations no longer appearing under their parent skills (#108, #97)
- The `linkedSkill` field on specializations stored a slug but matching was still done by name
- All lookups (character-sheet, roll-dialog, sheet-helpers, actor-uuid-resolver, dice-roller) now match by name **and** slug

### Narration actions incorrectly added to feats (#111)
- The old schema had `initial: 1` on `narrationActions`, adding +3 to the cost of every feat
- The migration resets `narrationActions` to 0 on feats where `grantsNarration` is `false`
- Fix applied to actors, world items **and** compendiums

### Validation errors on load
- **`featType: "vehicle" is not a valid choice`**: `FeatDataModel.migrateData()` automatically converts `"vehicle"` to `"equipment"` before Foundry validation
- **`type: "race" is not a valid type`**: `SRA2Actor.migrateData()` converts embedded items of type `"race"` to `"metatype"`, and `SRA2Item.migrateData()` does the same for world items

### Defense skill incorrectly pre-selected
- Defense against spells/complex forms/powers was selecting "Close Combat" instead of "Athletics"
- Fixed: magic attacks always default to Athletics (+ Spec: Ranged Defense) for defense
- Defense skill is now resolved via `findSkillByName()` which supports name, slug, and reverse mapping

### Errors in the EN compendium (#97)
- Slugs added to all skills and specializations in both EN and FR packs (182 files)

## New features

### Area of Effect for spells, complex forms, and powers (#95)
- New "Area of Effect (3m/zone)" field in the roll dialog for magic rolls
- Each zone adds +1 to each target's defense threshold
- Works with multi-targeting (Shift+click): single attack roll, individual comparison per target
- AoE is stored in message flags and applied during the defense roll

### Alphabetical sorting of feat lists (#110)
- All feat categories (weapons, spells, equipment, complex forms, etc.) are now sorted alphabetically

### Damage Value displayed under spells
- Indirect spells display their DV below the name: `DV: 6 (WIL+0)`
- Same style as weapon DV display

### Slug field on the specialization sheet
- The slug is now visible and editable on the specialization sheet (same as the skill sheet)

### Matrix counter-attack: priority-based selection
- Skill/spec selection for matrix defense and counter-attack now uses a priority cascade:
  1. Spec: Cybercombat (under Cracking)
  2. Spec: Complex Forms (under Technomancer)
  3. Cracking skill alone
  4. Technomancer skill alone
- No more binary techno/decker detection — based on what the character actually owns

## Technical notes
- Custom document classes: `SRA2Actor` and `SRA2Item` with `migrateData()` for backward compatibility
- Global cache `SRA2_SKILL_SLUG_CACHE` extended to include specializations (skills + specs)
- `findSkillByName()` supports normalized name, slug, and reverse `SKILL_NAME_TO_SLUG` mapping
- `findSpecBySlug()` and `findSkillBySlug()` in sra2-system.ts for matrix lookups
