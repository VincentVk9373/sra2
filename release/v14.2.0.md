# Release v14.2.0

## Feature — Risk Reduction on drones and vehicles

Vehicles are actors, so there was no way to model a drone that grants a Risk Reduction, like the security rigger's Lockheed surveillance drone (SRA2 p249). Vehicle sheets now accept any feat (not just weapons), and a new "Feats" list in the Weapons section shows each feat's RR entries with an active toggle.

How a drone feat's RR applies:

- **Targeted RR** (skill, specialization, or attribute) benefits the **drone owner's rolls**, like any other feat RR. The roll dialog shows the source suffixed with the drone's name (e.g. "Surveillance optics (Lockheed)"), so the table can judge whether the roll actually goes through the drone. If the owner lacks the skill, it also shows up as a phantom RR, as usual.
- **Untargeted (generic) RR** applies to the **drone's own rolls**: autopilot rolls, autonomous weapon attacks, and the owner's remote-controlled weapon rolls through that drone.

Details and fixes along the way:

- A weapon's generic RR no longer leaks onto the other weapons of the same vehicle.
- Fixed a double count of autopilot-targeted RR carried by an active vehicle weapon (counted once as an active feat, once as the weapon's own list).
- Drone RR sources are properly excluded from double counting when the drone weapon's own RR targets the skill used to control it.
