# Release v14.3.0

## Feature — RR tab on drone/vehicle sheets

Simpler take on the v14.2.0 drone RR feature, following feedback: no more dropping feats on drones. Vehicle sheets get a dedicated **RR tab** (same editor as feats: label, type, value, target with autocomplete search), stored on the vehicle itself. Vehicles are back to accepting weapons only.

- **Targeted RR** (skill, specialization, or attribute) benefits the **drone owner's rolls** once the drone is linked to the character sheet. The roll dialog shows the drone's name as the RR source, and the RR shows up as a phantom entry when the owner lacks the skill.
- **Untargeted RR** applies to the **drone's own rolls**: autopilot rolls, autonomous weapon attacks, and the owner's remote-controlled shots through that drone.

For the security rigger's Lockheed (SRA2 p249): open the drone sheet, RR tab, add an entry `RR1 → Perception` — done.

## Fix — Vehicle sheet layout and margins

The vehicle sheet's weapons block never received its intended styling (buttons, list spacing) due to a CSS selector mismatch, and section content sat flush against the sticky header. Sections now breathe: weapons, RR, narrative effects and description all get proper margins, and the description textarea has a usable size.
